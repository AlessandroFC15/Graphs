1ª Questão

A primeira questão da lista consistia em formular um procedimento que informasse a existência, ou não, de uma certa aresta de um determinado grafo, utilizando matriz e lista de adjacência. O procedimento permite escolher entre o grafo normal e o grafo completo e segue-se uma discussão sobre a performance do algoritmo.

Comando da Questão


Dado o grafo não direcionado G abaixo:



1) Elabore um procedimento que informe a existência, ou não, de uma certa aresta no grafo de entrada. A operação deve retornar “verdadeiro”, se a aresta (i, j) está presente no grafo, senão retornar “falso”.

Esse procedimento deve ser implementado usando matriz e lista de adjacência. Apresente log de execução tendo o grafo G como entrada.

Agora, transforme o grafo G em um grafo completo H. É possível perceber alguma diferença entre o tempo que o procedimento leva para encontrar arestas em H quando se compara matriz e lista? Explique.



Solução

O procedimento para informar a existência ou não existência de uma aresta no grafo G acima segue abaixo:

  • Lista de Adjacência

Procedimento

GrafoMatriz.prototype.existeAresta = function(verticeSaida, verticeChegada) {
    return this.matriz[verticeSaida][verticeChegada] != undefined &&
            this.matriz[verticeSaida][verticeChegada] != 0;
};
A função acima apenas checa se os valores passados como vértice de entrada e vértice de saída são valores válidos e em seguida verifica a existência da aresta no grafo, considerando que qualquer valor número que não seja 0 representará a existência de uma aresta. 
GrafoListaAdjacencia.prototype.existeAresta = function(verticeSaida, verticeChegada) {
    return this.lista[verticeSaida] &&
        this.lista[verticeSaida].indexOf(Number(verticeChegada)) != -1;
};
A função da implementação utilizando lista de adjacência é bem semelhante à função da matriz em relação aos passos executados. É verificado primeiramente a existência do vértice de saída e em seguida, analisamos se o vértice de chegada inserido está presente na lista de vértices adjacentes do vértice de saída.

Discussão

É possível perceber alguma diferença entre o tempo que o procedimento leva para encontrar arestas em H quando se compara matriz e lista? Explique.

Através de diversas execuções do algoritmo acima e com o auxílio da tabela de performance abaixo, podemos perceber que não existe uma diferença significativa na performance do procedimento acima comparando a implementação por lista e a implementação por matriz de adjacência.

Tal análise não deve ser uma surpresa, visto que os métodos de cada implementação são bem semelhantes, baseando-se em 2 testes, um para checar a validade dos dados de entrada e outro para realmente testar a presença ou não da aresta. É válido ressaltar enquanto a verificação da existência da aresta na matriz é direta, isto é, verificar se um valor corresponde a 0 ou não, no caso da implementação por lista, é necessário uma função auxiliar.

O método auxiliar é o indexOf, que no caso da implementação acima tem como função procurar a existência do vértice de chegada dentro da lista de vértices adjacentes do vértice de saída. Tal procedimento é obviamente mais custoso do que um simples teste, como ocorre no procedimento do grafo matriz.

Para o nosso caso, onde o número de vértices não é elevado, tal diferença entre a performance dos procedimentos não foi significativa. No entanto, caso o número de vértices seja bem mais elevado, é de se esperar que a implementação por matriz seja mais eficiente, levando em conta o seu acesso direto.
